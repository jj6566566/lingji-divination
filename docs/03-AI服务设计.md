## "引擎 + 解释器" 架构（核心设计）

```
                    算卦引擎 (纯代码，不调 LLM)
                    ════════════════════════
                    负责: 生成卦象、掷铜钱、排八字、抽牌
                    特点: 确定性、可复现、零 Token 成本

        六爻引擎   八字引擎   塔罗引擎   梅花易数引擎
           │          │          │          │
           └──────────┼──────────┼──────────┘
                      │
                      ▼
                统一解卦解释器 (调 LLM)
                ════════════════════
                负责: 理解卦象 + 用户问题 → 流式解读
                特点: 动态加载 System Prompt、知识库、上下文
```

**为什么不是多个 Agent？** 新增占卜方式只需：加一个引擎 + 加一个 Prompt 模板 + 注册。上下文管理、压缩、会话全部自动继承。

## 核心逻辑：一次占卜的处理流程

```python
async def divine_stream(method, question, session_id, context):
    # 第1步: 算卦（纯代码，不花 token）
    engine = engine_registry.get(method)
    result = engine.cast()  # 返回卦象 dict

    # 第2步: 查知识库
    knowledge = knowledge_loader.lookup(method, result)
    # 六爻 → 匹配卦辞爻辞
    # 八字 → 匹配十神分析规则
    # 塔罗 → 匹配正逆位含义

    # 第3步: 获取会话上下文（如果有历史对话）
    session = session_manager.get_or_create(session_id)

    # 第4步: 组装最终 Prompt
    system_prompt = prompt_manager.load(method)
    assembled = context_assembler.build(
        system_prompt=system_prompt,     # 层级1: 角色指令
        knowledge=knowledge,              # 层级2: 卦象知识
        result=result,                    # 层级2: 算卦结果
        question=question,                # 用户问题
        user_context=context,             # 用户画像
        chat_history=session.get_history()# 层级3: 追问历史
    )

    # 第5步: 流式调用 Claude
    async for chunk in llm_client.stream(assembled):
        yield chunk  # 逐句推给 Java 后端

    # 第6步: 更新会话历史
    session.append("user", question)
    session.append("assistant", full_response)

    # 第7步: 检查是否需要压缩
    if session.token_count() > 8000:
        session.compress()
```

## 上下文压缩策略

### 触发条件
```
当前上下文 token 数 > 8000 → 触发压缩
保留最近 5 轮原始对话
第 5 轮之前的 → 压缩为结构化摘要
```

### 压缩内容（调 LLM 做摘要）

```
输入: 第 1~N 轮的原始对话
输出: 结构化摘要 JSON

{
  "user_profile": "男，33岁，程序员，妻子支持跳槽",
  "divination_brief": "首次占得地天泰䷊→雷天大壮䷡，九二变爻，核心解读: 跳槽机会存在，春季有利",
  "key_insights": [
    "AI建议明年2-4月是最佳窗口",
    "用户倾向跳去大厂而非创业"
  ],
  "unresolved_questions": ["具体去哪家还没定"]
}
```

### 为什么用结构化摘要？
- Token 消耗比原文少 80%
- AI 检索精准，不支持"猜"
- 结构化数据可以喂给前端做"历史回顾"功能

## 算卦引擎接口

所有引擎实现同一个基类：

```python
class BaseEngine(ABC):
    """算卦引擎基类"""
    @abstractmethod
    def cast(self, **params) -> dict:
        """执行占卜，返回统一格式的卦象 dict"""
        pass

    @abstractmethod
    def validate_params(self, **params) -> bool:
        """校验参数（如八字需要生日）"""
        pass

    @property
    @abstractmethod
    def method_name(self) -> str:
        """返回占卜方式标识: liuyao / bazi / tarot"""
        pass
```

### 六爻引擎示例

```python
class LiuyaoEngine(BaseEngine):
    method_name = "liuyao"

    def cast(self, **params) -> dict:
        lines = []
        for i in range(6):
            # 三枚铜钱，正面=2，反面=3
            coins = [random.randint(0, 1) for _ in range(3)]
            value = sum(2 if c == 1 else 3 for c in coins)
            lines.append({
                "position": i + 1,
                "value": value,         # 6/7/8/9
                "type": self._judge_type(value),
                "changing": value in (6, 9),
            })

        return {
            "lines": lines,
            "original": self._to_hexagram(lines),
            "transformed": self._to_transformed(lines),
            "changing_lines": [l for l in lines if l["changing"]],
        }
```

## Prompt 模板示例

### liuyao_system.md

```markdown
你是精通周易六爻预测的占卜师。你博学、沉稳、有共情力。

## 你的风格
- 先讲卦象本身含义，再结合用户问题具体分析
- 有理有据：引用卦辞、爻辞原文，再翻译成白话
- 积极但有分寸：给予希望，但不妄下绝对判断
- 卦是参考，最终选择权在用户自己
- 用户是中国人，用中文对话，语言优雅简洁

## 你的约束
- 不回答与本次占卜无关的问题
- 不声称能预测具体数字（彩票、股价等）
- 不替代医疗、法律等专业建议
- 如果用户问题包含违法内容，温和拒绝
- 不需要逐字翻译原文，重点是让用户理解

## 当前卦象
{result}

## 相关卦辞
{knowledge}

## 用户情况
{user_context}

现在请为用户解读此卦。
```

## 模型选择

| 场景 | 模型 | 理由 |
|------|------|------|
| 正式解读 | claude-sonnet-5 | 中文最好，性价比最高 |
| 摘要压缩 | claude-haiku-4-5 | 压缩是机械任务，用便宜的 |
| 以后需要深度推理 | claude-opus-4-8 | 复杂八字命理可能需要 |
