# 灵机 — AI 智能占卜 SaaS 架构设计

## 产品定位

AI 驱动的多方式占卜平台。用户选择占卜方式 → 起卦 → AI 解读 → 追问。

## 三层架构

```
用户 (浏览器/微信)
        │
        ▼
┌───────────────────────────────────────────┐
│           Nginx 反向代理 (生产环境)          │
│           域名、HTTPS、负载均衡              │
└──────────────┬────────────────────────────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐
│ 前端    │ │Java 后端│ │AI 服务  │
│Next.js │ │Spring  │ │FastAPI │
│ :3000  │ │:8080   │ │ :8000  │
└────────┘ └───┬────┘ └────────┘
               │
          ┌────┴────┐
          ▼         ▼
      ┌──────┐ ┌──────┐
      │MySQL │ │Redis │
      │ 8.0  │ │7.x   │
      └──────┘ └──────┘
```

## 各层职责

| 层 | 语言 | 框架 | 负责 | 不该做 |
|----|------|------|------|--------|
| 前端 | TypeScript | Next.js 14 | UI、交互、动效、流式接收 | 算卦逻辑、直接调 LLM |
| 后端 | Java | Spring Boot 3 | 用户、鉴权、记录、配额、调 AI 服务 | 起卦、直接调 LLM |
| AI | Python | FastAPI | 起卦算法、Prompt 组装、LLM 调用、上下文管理 | 用户管理、鉴权 |

## 数据流（一次占卜）

```
前端 POST /api/v1/divination {method:"liuyao", question:"..."}
  → Java 鉴权 → 查配额 → 调 AI 服务 → SSE 流式转发给前端
  → AI 服务: 起卦(纯算法) → 查知识库 → 组装 Prompt → 调 Claude 流式返回
  → 解读结束 → Java 存库 → 配额更新
```

## 技术栈决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 前端 | Next.js 14 | AI SDK 生态、SSR、就业最好 |
| UI 库 | shadcn/ui | 复制即用、Tailwind 原生 |
| 后端语言 | Java 21 | 你学这个，企业级特性 |
| 后端框架 | Spring Boot 3 | Java 标配 |
| ORM | MyBatis-Plus | 国内主流、灵活 |
| 鉴权 | JWT | 无状态、前后端分离标准 |
| AI 框架 | FastAPI | 异步、自动生成文档 |
| LLM | Claude API | 中文能力最强 |
| 向量库 | Chroma (V2) | Python 原生、零配置 |
| 数据库 | MySQL 8.0 | 结构化数据标配 |
| 缓存 | Redis 7 (V2) | 限流、Session |
| 部署 | Docker Compose | 一键启动 |
